Модуль приёма задач, способный работать с несколькими источниками через единый контракт. 

Источники задач: из файла, генерируемые программно и задачи, получаемые из API.
Возможно добавление новых источников задач без изменения текущего кода. 
Все источники задач реализуют единый контракт, описанный через typing.Protocol,
при помощи метода get_tasks(). Реализованы несколько runtime-проверок соблюдения контракта
и его методов. Написаны тесты, проверяющие правильность валидации контрактов и их соблюдения.

Источники задач принимают путь до файла с задачами и возвращают список задач, реализованных через dataclass.

Модель задачи Task() проверяет входные данные через дескрипторы и @property
и при неверных данных поднимает кастомные ошибки `TaskValidationError` и `InvalidTaskStateError`


Реализована очередь задач `TaskQueue` с поддержкой протокола итерации, повторного обхода и ленивых фильтров (по статусу, приоритету, пользовательскому предикату) через генераторы. Очередь обеспечивает потоковую обработку задач с автоматическим изменением статусов (`PENDING` → `IN_PROGRESS` → `COMPLETED/FAILED`), индексацию и срезы, а также методы анализа (`count`, `any`, `all`). Все операции ленивы и не создают избыточных копий данных, что позволяет работать с большими объёмами задач. Код полностью интегрирован с модулем приёма задач (источники из файла, API, генератор) и моделью `Task`, протестирован с покрытием более 80%.

Запуск программы 
```shell
python3 -m main.py
```

Запуск тестов 
```shell
pytest --cov=src tests/
```
